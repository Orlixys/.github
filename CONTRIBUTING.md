# Guia de Contribuição - Orlixys

Obrigado por se interessar em contribuir com a Orlixys! Como uma organização focada em segurança, seguimos padrões rigorosos.

## Regras de Ouro
1. **Signed Commits:** Todos os commits devem ser assinados digitalmente (GPG/SSH). Commits não assinados serão rejeitados automaticamente.
2. **Pull Requests:** Nenhuma alteração entra na `main` sem um Pull Request revisado.
3. **Qualidade de Código:** Siga os padrões de linting e formatação definidos em cada projeto.

## Como Contribuir?
1. Faça um **Fork** do repositório.
2. Crie uma **Branch** para sua alteração (`git checkout -b feature/minha-melhoria`).
3. Faça o commit das suas mudanças (assine o commit!).
4. Envie para o seu fork e abra um **Pull Request**.

## Código de Conduta
Mantenha a cordialidade. Estamos aqui para construir uma tecnologia segura e aberta para todos.
